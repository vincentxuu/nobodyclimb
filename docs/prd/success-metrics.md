## 1. 指標框架

### 1.1 北極星指標 (North Star Metric)

**定義**: 最能代表產品核心價值的單一指標

**NobodyClimb 的北極星指標**:
```
每週活躍內容創作者數 (Weekly Active Content Creators)
```

**選擇理由**:
- 內容創作者是平台的核心價值驅動者
- 優質內容吸引更多使用者
- 創作者活躍度反映平台健康度
- 與長期增長和留存直接相關

**計算方式**:
```
WACC = 過去7天內發布至少1篇內容（文章/照片/評論）的獨立使用者數
```

### 1.2 AARRR 指標體系

基於海盜指標 (Pirate Metrics) 框架：

```
Acquisition (獲取) → Activation (啟動) → Retention (留存)
  → Revenue (營收) → Referral (推薦)
```

### 2.2 Activation (啟動)

#### 新使用者啟動

**啟動定義**: 新使用者在註冊後7天內完成以下至少 3 項行為：
1. 完成個人檔案設置
2. 瀏覽至少 3 篇文章
3. 收藏至少 1 個場館或內容
4. 追蹤至少 1 位使用者
5. 發表至少 1 則評論

| 指標 | 定義 | 目標 (MVP) | 目標 (1年後) |
|------|------|-----------|------------|
| **啟動率** | 啟動使用者 / 註冊使用者 | 40% | 60% |
| **首次互動時間** | 註冊到首次互動的時間 | < 5 分鐘 | < 3 分鐘 |
| **檔案完成率** | 完成檔案設置 / 註冊使用者 | 60% | 80% |

**關鍵行為漏鬥**:
```
註冊完成 (100%)
  ↓
開始檔案設置 (80%)
  ↓
完成檔案設置 (60%)
  ↓
首次內容瀏覽 (70%)
  ↓
首次互動（點讚/評論/收藏） (40%)
  ↓
啟動成功 (40%)
```

### 3.3 使用者滿意度

| 指標 | 定義 | 目標 | 測量方式 |
|------|------|------|---------|
| **NPS** (淨推薦值) | 推薦意願評分 | > 40 | 季度調查 |
| **CSAT** (客戶滿意度) | 整體滿意度 | > 4.0/5.0 | 功能使用後調查 |
| **CES** (客戶費力度) | 完成任務難易度 | < 3.0/7.0 | 關鍵流程後調查 |

**NPS 計算**:
```
NPS = % 推廣者 (9-10分) - % 貶損者 (0-6分)
```

**滿意度調查觸發點**:
- 註冊後 7 天
- 發布首篇文章後
- 使用 30 天後
- 重大功能更新後

## 7. 數據追蹤方法

### 7.1 數據收集工具

**前端分析**:
- **Google Analytics 4**: 使用者行為追蹤
- **Cloudflare Analytics**: 流量和效能
- **Hotjar** (未來): 熱點圖和錄影

**後端分析**:
- **Django Logger**: 應用程式日誌
- **PostgreSQL Query Analysis**: 數據庫效能
- **Sentry** (未來): 錯誤追蹤

### 7.2 事件追蹤架構

**事件命名規範**:
```
{對象}_{動作}_{結果}

範例:
- article_view_success
- user_register_start
- user_register_complete
- gym_bookmark_add
- comment_post_success
```

**關鍵事件列表**:

**使用者旅程事件**:
```javascript
// 註冊流程
track('user_register_start');
track('user_register_email_entered');
track('user_register_password_created');
track('user_register_complete');
track('user_register_email_verified');

// 檔案設置
track('profile_setup_start');
track('profile_setup_step1_complete');  // 基本資訊
track('profile_setup_step2_complete');  // 攀巖經驗
track('profile_setup_complete');

// 內容互動
track('article_view', { article_id, category });
track('article_like', { article_id });
track('article_bookmark', { article_id });
track('comment_post', { article_id, comment_length });

// 場館互動
track('gym_view', { gym_id, region });
track('gym_bookmark', { gym_id });
track('gym_review_post', { gym_id, rating });
```

### 7.3 使用者屬性

**基本屬性**:
- user_id
- registration_date
- user_type (新手/進階/達人)
- location (地區)
- preferred_language

**行為屬性**:
- total_posts
- total_comments
- total_bookmarks
- last_active_date
- activity_level (低/中/高/超級)
