# 人物誌填寫體驗優化規劃

> 文件版本：v1.0
> 建立日期：2026-01-18
> 狀態：規劃中

---

## 1. 背景與問題陳述

### 1.1 現況描述

目前人物誌系統採用三層式設計：

| 層級 | 內容 | 題數 | 必填性 |
|-----|------|-----|--------|
| 第一層 | 基本資訊（開始年份、常去地點、路線類型） | 3 題 | 必填 |
| 第二層 | 核心故事（相遇、意義、建議） | 3 題 | 強烈建議 |
| 第三層 | 進階故事（6 大分類） | 31 題 | 選填 |

**總計：37 個欄位，其中 34 題為開放性文字填寫**

### 1.2 核心問題

用戶反映「填寫人物誌感覺很困難」，經分析後歸納出以下原因：

#### A. 數量壓力 (Quantity Overload)

- 34 個開放性問題一次呈現
- 進度顯示 `0/31` 產生挫敗感
- 用戶不知從何開始

#### B. 問題設計門檻過高

- **抽象/哲學性問題**：需要深度反思
  - 「攀岩對你來說是什麼」
  - 「攀岩如何改變你看待失敗」
  - 「在岩壁上的心流時刻」

- **需要深度回憶**：需回想具體情境
  - 「最難忘的攀登經歷」
  - 「第一次戶外攀岩的經歷」

#### C. 純文字填寫門檻

- 全部採用 Textarea 開放輸入
- 缺乏選項、多選、評分等低門檻互動方式
- 對不擅長文字表達的用戶造成障礙

#### D. 範例造成完美主義壓力

現有 placeholder 過於完整：
```
「2022 年的秋天，朋友帶我去岩館，第一次嘗試就愛上了
那種專注在每一個動作的感覺...」
```

用戶可能因此產生「我寫不出這麼好的文字」的心理障礙。

#### E. 缺乏參考與引導

- 沒有其他用戶的填寫範例
- 缺乏對話式或分步驟引導流程
- 不確定「正確」的內容長度與深度

---

## 2. 目標設定

### 2.1 核心目標

**降低人物誌填寫的心理門檻，提升完成率與用戶滿意度**

### 2.2 量化指標

| 指標 | 現況（預估） | 目標 |
|-----|-------------|------|
| 核心故事（3題）完成率 | ~30% | 60% |
| 進階故事至少填 1 題比例 | ~15% | 40% |
| 平均填寫題數 | ~2 題 | ~5 題 |
| 填寫流程放棄率 | 高 | 降低 50% |

---

## 3. 解決方案

### 3.1 Phase 1：快速優化（1-2 天）

#### 3.1.1 降低範例壓力

**現狀：**
```typescript
placeholder="例如：2022 年的秋天，朋友帶我去岩館，第一次嘗試就愛上了那種專注在每一個動作的感覺..."
```

**改為：**
```typescript
placeholder="例如：「朋友帶我去試，結果一爬就愛上了」"
// 或更簡短
placeholder="寫下你的故事，長短都可以..."
```

**影響檔案：**
- `src/lib/constants/biography-stories.ts`
- `src/components/profile/ClimbingExperienceSection.tsx`

#### 3.1.2 優化進度顯示語言

**現狀：**
```
0/31 完成
```

**改為：**
```
已收藏 0 個故事 ✨
或
開始記錄你的第一個故事吧！
```

**影響檔案：**
- `src/components/profile/AdvancedStoriesSection.tsx`

#### 3.1.3 新增明確跳過提示

在編輯區域加入提示文字：

```
💡 所有故事都是選填的，想到什麼就寫什麼，隨時可以跳過
```

**影響檔案：**
- `src/components/profile/AdvancedStoriesSection.tsx`
- `src/components/profile/ClimbingExperienceSection.tsx`

#### 3.1.4 新增「隨機推薦一題」功能

在進階故事區塊加入按鈕：

```
[🎲 不知道寫什麼？隨機挑一題]
```

點擊後從未填寫的題目中隨機選一題並展開編輯。

**影響檔案：**
- `src/components/profile/AdvancedStoriesSection.tsx`
- `src/components/biography/advanced-story-editor.tsx`（如存在）

### 3.2 Phase 2：體驗重設計（1-2 週）

#### 3.2.1 選項+自由輸入混合模式

將部分問題改為先選擇、後補充的形式：

**範例：「攀岩教會你最重要的一件事」**

```
攀岩教會我最重要的是：
○ 面對恐懼
○ 保持專注
○ 接受失敗
○ 相信自己
○ 其他：______

想多說一點嗎？（選填）
[                    ]
```

**優點：**
- 降低「不知道怎麼開始」的障礙
- 用戶可以只選選項就完成
- 想寫的人仍可補充

#### 3.2.2 簡化版問題

為每個問題提供「簡單版」替代：

| 原問題 | 簡化版 |
|-------|--------|
| 攀岩對你來說是什麼 | 你最喜歡攀岩的哪個部分？ |
| 在岩壁上的心流時刻 | 有沒有爬到忘記時間的經驗？ |
| 攀岩如何改變你看待失敗 | 爬不上去的時候你會怎麼想？ |

#### 3.2.3 對話式填寫引導

新增「引導填寫」模式，一次只問一題：

```
┌─────────────────────────────────┐
│  💬 來聊聊你的攀岩故事吧！        │
│                                 │
│  你是哪一年開始攀岩的？           │
│                                 │
│  [2020] [2021] [2022] [2023] [更早] │
│                                 │
│  [下一題 →]              [稍後再說] │
└─────────────────────────────────┘
```

**特點：**
- 一次只顯示一題，減少認知負擔
- 使用按鈕/選項降低輸入門檻
- 可隨時中斷，下次繼續

#### 3.2.4 範例展示功能

展示其他岩友的回答作為參考：

```
┌─────────────────────────────────┐
│  📖 看看其他岩友怎麼寫            │
│                                 │
│  「第一次去龍洞，被風景震撼到，    │
│   雖然只爬了 5.8 但超開心」       │
│                           — 小明  │
│                                 │
│  「朋友硬拉我去，結果比他還認真」  │
│                           — 阿花  │
└─────────────────────────────────┘
```

**注意事項：**
- 需取得用戶授權或使用匿名
- 可由團隊先撰寫範例內容

### 3.3 Phase 3：長期策略（1-3 個月）

#### 3.3.1 分階段漸進收集

不要一次要求填完，改為分階段推送：

| 時機 | 收集內容 |
|-----|---------|
| 註冊完成時 | 基本資訊 3 題 |
| 第一週 | 推送「本週故事提問」1 題 |
| 每週/每月 | 持續推送新問題 |
| 特定時機 | 完攀新難度時自動詢問 |

#### 3.3.2 社群激勵機制

- **徽章系統**：填完特定題目解鎖徽章
- **朋友動態**：「你的朋友 OOO 新增了一個故事」
- **完成度排行**：展示社群平均完成度

#### 3.3.3 AI 輔助填寫

提供 AI 助手幫助用戶：

```
┌─────────────────────────────────┐
│  🤖 需要幫忙嗎？                  │
│                                 │
│  你可以先說幾個關鍵字，           │
│  我幫你整理成完整的句子           │
│                                 │
│  [試試 AI 助寫]                  │
└─────────────────────────────────┘
```

**功能：**
- 從關鍵字擴寫成段落
- 潤稿/修正文字
- 建議寫作方向

#### 3.3.4 題目精簡

考慮將 31 題進階故事精簡為 **10-15 題精選**：

**保留原則：**
1. 回答門檻較低的題目
2. 展示效果好的題目
3. 社群互動價值高的題目

**可考慮移除或合併：**
- 過於相似的題目（如「最大突破」與「最大挑戰」）
- 回答難度極高的題目

## 6. 風險與注意事項

| 風險 | 影響 | 緩解措施 |
|-----|-----|---------|
| 過度簡化導致內容品質下降 | 人物誌展示效果變差 | A/B 測試、保留原有選項 |
| AI 助寫產生同質化內容 | 人物誌失去個人特色 | 限制 AI 參與程度、強調個人化 |
| 題目精簡引發既有用戶不滿 | 已填寫的內容被移除 | 保留資料、提供遷移方案 |
| 範例展示侵犯隱私 | 用戶投訴 | 明確授權機制、匿名處理 |
