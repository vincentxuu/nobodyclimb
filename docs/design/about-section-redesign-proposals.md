## 方案一：左右分欄式 (Split Layout)

**參考**：Apple、Stripe、Notion 官網

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  ┌──────────────────────┐    ┌──────────────────────────────┐  │
│  │                      │    │                              │  │
│  │     [攀岩照片]        │    │   關於小人物攀岩              │  │
│  │                      │    │   ─────────────              │  │
│  │   可放置品牌形象圖     │    │                              │  │
│  │   或社群精選照片       │    │   緣起於一個 Nobody 很喜歡   │  │
│  │                      │    │   這項運動，希望有更多        │  │
│  │                      │    │   Nobody 也能一起來 Climb    │  │
│  │                      │    │                              │  │
│  │                      │    │   • 分享攀岩知識              │  │
│  │                      │    │   • 連結攀岩社群              │  │
│  │                      │    │   • 記錄攀岩故事              │  │
│  │                      │    │                              │  │
│  │                      │    │   [認識小人物] [成為小人物]   │  │
│  │                      │    │                              │  │
│  └──────────────────────┘    └──────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**設計特點**：
- 左側 50%：大尺寸圖片或影片
- 右側 50%：文字內容靠左對齊
- 可加入小型列點說明品牌價值
- 手機版上下堆疊

**優點**：
- 視覺層次分明
- 圖片更有存在感
- 文字易讀性高

**技術細節**：
- Grid 或 Flexbox 兩欄佈局
- 圖片可用 `object-fit: cover`
- 支援反轉版本（圖右文左）

## 方案二：數據統計展示 (Stats Showcase)

**參考**：Airbnb、Spotify Wrapped、GitHub

```
┌─────────────────────────────────────────────────────────────────┐
│                         淺色背景                                 │
│                                                                 │
│                      關於小人物攀岩                              │
│         緣起於一個 Nobody 很喜歡這項運動...                      │
│                                                                 │
│  ┌───────────┐    ┌───────────┐    ┌───────────┐               │
│  │           │    │           │    │           │               │
│  │   500+    │    │   50+     │    │   1000+   │               │
│  │  攀岩影片  │    │  岩場介紹  │    │  社群成員  │               │
│  │           │    │           │    │           │               │
│  └───────────┘    └───────────┘    └───────────┘               │
│                                                                 │
│                     [認識小人物]                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**設計特點**：
- 用數據展示社群規模與影響力
- 數字使用大字體 + 動畫計數效果
- 簡潔乾淨的視覺風格

**優點**：
- 建立信任感與權威性
- 視覺衝擊力強
- 讓訪客感受社群活力

**可展示數據**：
- 影片數量
- 岩場數量
- 人物誌數量
- 社群成員數
- 文章數量

**技術細節**：
- 數字滾動動畫 (Framer Motion)
- Intersection Observer 觸發
- 數據可從 API 動態取得

**技術細節**：
- CSS 偽元素繪製時間軸
- Scroll-driven animation
- 可展開查看更多細節

## 方案比較表

| 方案 | 視覺衝擊 | 資訊量 | 開發複雜度 | 適合情境 |
|------|---------|--------|-----------|---------|
| 一、左右分欄 | ⭐⭐⭐ | ⭐⭐⭐ | 低 | 經典穩妥，適合大多數情況 |
| 二、數據統計 | ⭐⭐⭐⭐ | ⭐⭐ | 中 | 強調社群規模與成就 |
| 三、時間軸 | ⭐⭐⭐ | ⭐⭐⭐⭐ | 中 | 強調品牌故事與發展 |
| 四、影片背景 | ⭐⭐⭐⭐⭐ | ⭐ | 中高 | 追求視覺震撼，需有影片素材 |
| 五、卡片網格 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 低 | 功能導向，引導探索 |

---

## 建議

根據小人物攀岩的定位（攀岩社群平台），建議考慮：

1. **方案一（左右分欄）** - 如果想要簡潔有質感
2. **方案二（數據統計）** - 如果想展示社群成長
3. **方案五（卡片網格）** - 如果想引導用戶探索功能

也可以**混合設計**，例如：
- 方案一 + 方案二：左右分欄，右側包含數據統計
- 方案五 + 方案一：上方簡介 + 下方功能卡片
