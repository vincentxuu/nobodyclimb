## 背景與目標

### 背景
當使用者在人物誌列表、文章頁面、評論區等位置看到作者頭像時，點擊後應能快速了解該作者的基本資訊，建立社群連結感。

### 目標
1. **提升使用者黏著度**：讓讀者更了解內容創作者，增加互動意願
2. **建立社群連結**：促進攀岩愛好者之間的交流
3. **鼓勵內容創作**：透過曝光機制激勵使用者撰寫更多內容
4. **保持一致體驗**：與現有人物誌、個人檔案系統整合

## 使用情境

### 進入點 (Entry Points)

| 位置 | 說明 |
|------|------|
| 文章頁面 | 文章標題下方的作者資訊區塊 |

### 使用者故事

```
作為一個攀岩愛好者
當我閱讀一篇有趣的攀岩文章時
我想要點擊作者頭像了解更多關於作者的資訊
以便判斷是否追蹤這位作者或閱讀更多他的內容
```

```
作為一個內容創作者
我希望其他人點擊我的頭像時
能看到我的攀岩背景和創作內容
以便吸引更多讀者關注我的文章
```

## 競品分析

### Medium
- **特色**：簡潔個人頁面、文章列表、追蹤者數量
- **優點**：乾淨、專注於內容
- **缺點**：個人化程度較低

### 方格子 (Vocus)
- **特色**：作者專頁、付費訂閱、社交連結
- **優點**：創作者經濟模式完整
- **缺點**：功能較複雜

### 少數派
- **特色**：作者介紹、貢獻統計、社群互動
- **優點**：社群感強、激勵機制
- **缺點**：資訊量較多

### 知乎
- **特色**：懸停卡片預覽、完整個人頁、回答/文章分類
- **優點**：快速預覽體驗佳
- **缺點**：資訊密度高

## 版本 A：簡約快速預覽版

### 概述
使用懸停 (Hover) 或點擊後彈出的小型卡片，讓使用者快速瀏覽作者基本資訊，無需離開當前頁面。

### 設計理念
- **最小干擾**：不打斷閱讀體驗
- **快速瀏覽**：3 秒內獲取關鍵資訊
- **輕量互動**：簡單的追蹤/查看更多操作

### 呈現方式
彈出式卡片 (Popover Card)

### 卡片內容

```
┌─────────────────────────────────────────┐
│  ┌──────┐                               │
│  │ 頭像 │  攀岩者名稱                    │
│  │ 80px │  @username                    │
│  └──────┘                               │
│                                         │
│  🧗 攀岩 5 年 ｜ 📍 北部岩場              │
│                                         │
│  「攀岩是我認識自己的方式」               │
│                                         │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  │
│  │ 3 篇文章 │  │ 人物誌  │  │ 12 追蹤 │  │
│  └─────────┘  └─────────┘  └─────────┘  │
│                                         │
│  [ 追蹤 ]          [ 查看完整檔案 → ]    │
└─────────────────────────────────────────┘
```

### 資訊欄位

| 欄位 | 說明 | 資料來源 |
|------|------|----------|
| 頭像 | 使用者頭像，圓形 80x80px | User.avatar |
| 名稱 | 顯示名稱 | User.name / Biography.name |
| 帳號 | @username | User.username |
| 攀岩年資 | 計算自開始年份 | Biography.startYear |
| 常去區域 | 簡化顯示 | Biography.frequentGyms |
| 一句話簡介 | 核心理念或自我介紹 | Biography.climbingMeaning (截取) |
| 文章數 | 已發表文章數量 | Article count |
| 人物誌狀態 | 是否有人物誌 | Biography.isPublic |
| 追蹤數 | 追蹤者數量 | Follow count |

### 互動功能

| 功能 | 說明 |
|------|------|
| 追蹤/取消追蹤 | 快速追蹤按鈕 |
| 查看完整檔案 | 連結到作者頁面 |
| 頭像點擊 | 連結到作者頁面 |

### 觸發方式
- **桌面版**：滑鼠懸停 300ms 後顯示，移出 200ms 後隱藏
- **行動版**：點擊頭像彈出，點擊外部關閉

### 技術實作建議

```typescript
// 組件結構
interface AuthorPreviewCardProps {
  userId: string;
  trigger: React.ReactNode;  // 觸發元素（頭像）
}

// 使用 Radix UI HoverCard 或 Popover
import { HoverCard } from '@radix-ui/react-hover-card';
```

### 優點
- 實作成本低
- 不影響現有頁面結構
- 使用者體驗流暢

### 缺點
- 資訊量有限
- 無法展示完整內容歷史
- 行動端體驗需額外優化

### 路由設計

```
/author/[username]           # 作者主頁（預設顯示文章）
/author/[username]/articles  # 文章列表
/author/[username]/biography # 人物誌內容
/author/[username]/about     # 關於作者
```

### 技術實作建議

```typescript
// 頁面結構
src/app/author/
  ├── [username]/
  │   ├── page.tsx           // 主頁面
  │   ├── layout.tsx         // 共用 layout（頭部資訊）
  │   ├── articles/page.tsx  // 文章列表
  │   ├── biography/page.tsx // 人物誌
  │   └── about/page.tsx     // 關於

// API 端點
GET /api/v1/users/:username/public-profile
GET /api/v1/users/:username/articles
GET /api/v1/users/:username/biography
```

### 與現有系統整合

| 現有頁面 | 整合方式 |
|----------|----------|
| `/profile` | 私人設定頁面，編輯資料 |
| `/author/[username]` | 公開展示頁面，只讀 |
| `/biography/profile/[id]` | 可重導向到 `/author/[username]#biography` |

### 優點
- 完整展示作者資訊
- 有利於 SEO
- 可分享的個人頁面連結
- 建立創作者個人品牌

### 缺點
- 開發成本較高
- 需要新增後端 API
- 需考慮與現有 profile/biography 頁面的關係
