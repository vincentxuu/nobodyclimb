# 冷啟動執行手冊（Cold Start Playbook）

> **目標**：解決「看得人多，寫的人少」的雞生蛋問題
> **資源盤點**：核心攀岩者人脈 ✓ 攀岩社群/社團 ✓ 岩館合作 ✓ 媒體/KOL ✓

## 二、種子用戶策略（Phase 0）

### 2.1 種子用戶定義

**不是找「最厲害的人」，是找「最願意寫的人」**

```
理想種子用戶畫像：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  ✓ 本來就喜歡寫（有 Blog、常發 FB 長文）                    │
│  ✓ 有故事可講（攀岩經歷豐富、有獨特經驗）                   │
│  ✓ 願意幫忙（認同平台理念、人情好說話）                     │
│  ✓ 有影響力（朋友多、會分享）                               │
│                                                             │
│  不一定是最強的攀岩者，但一定是有故事的人                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 種子用戶數量目標

```
第一批：20-30 人
├─ 10 人：核心故事完整（3 題以上）
├─ 10 人：主動回應他人故事（3 則以上）
└─ 10 人：願意分享到社群（FB/LINE）

為什麼是 20-30 人？
→ 足夠讓新用戶進來不會覺得空
→ 可以形成最小的社群互動
→ 你可以一個一個邀請、一個一個輔導
```

### 2.3 邀請話術

**錯誤示範**：
```
「欸我做了一個攀岩網站，你來註冊一下」
→ 用戶會想：又一個網站，我又沒差
```

**正確示範**：
```
「我在做一個攀岩社群平台，想收集攀岩者的故事。
 我覺得你的故事很有意思，想邀請你當第一批用戶。
 不用寫很多，就回答三個問題：
 1. 你是怎麼開始攀岩的
 2. 攀岩對你的意義
 3. 給剛開始的自己一句話

 你願意幫忙嗎？」
```

**要點**：
- 強調「你的故事很特別」→ 解決「我沒什麼好寫」
- 明確說只要「三個問題」→ 降低門檻
- 問「願意幫忙嗎」→ 利用互惠心理

### 2.4 種子用戶輔導

**不要只是邀請，要輔導到完成**

```
Day 1：發送邀請
└─ 「你願意幫忙嗎？」

Day 2：確認註冊
└─ 「註冊順利嗎？有任何問題可以問我」

Day 3：確認填寫
└─ 「看到你的故事了！寫得很好，有共鳴」
└─ 如果還沒寫：「需要幫忙想怎麼開頭嗎？」

Day 5：請求分享
└─ 「可以把你的人物誌分享到 FB 嗎？」

Day 7：請求回應別人
└─ 「小美也寫了她的故事，你要不要去回應一下？」
```

| 姓名 | 邀請日期 | 註冊 | 核心故事 | 回應他人 | 分享 | 收到回應 |

## 三、內容種子策略

### 3.1 確保「每個故事都有回應」

**最大的冷啟動風險**：用戶寫了但沒人理

```
策略：在種子期間，確保每一個故事都有至少 2-3 則回應

執行方式：
1. 你自己回應（用個人帳號，不是官方帳號）
2. 請種子用戶互相回應
3. 有選擇性地推播「新故事」給其他種子用戶

目標：讓每個寫故事的人都在 24 小時內收到回應
```

### 3.2 內容多樣性

**不要全部都是「厲害的人」**

```
內容組成目標：

故事類型分布：
├─ 30% 普通週末攀岩者（降低門檻）
├─ 30% 有趣經歷者（吸引閱讀）
├─ 20% 進階攀岩者（增加可信度）
└─ 20% 新手/初學者（讓新手覺得「我也可以」）

地區分布：
├─ 北部岩場（龍洞、大砲岩等）
├─ 中部岩場
├─ 南部岩場
└─ 室內岩館

攀岩類型：
├─ 抱石
├─ 運動攀登
├─ 傳統攀登
└─ 室內為主
```

### 3.3 精選內容策劃

**首頁要展示「最好的內容」**

```
精選標準：
✓ 故事有趣、有共鳴點
✓ 文字流暢、有畫面
✓ 能代表「普通人也有故事」
✓ 有照片更好

首頁展示策略：
┌─────────────────────────────────────────────────────────────┐
│  首頁 Banner                                                 │
│  「每個攀岩者都有故事，包括你」                              │
│                                                             │
│  精選故事 1：有趣的入坑故事（引起共鳴）                      │
│  精選故事 2：克服恐懼的故事（激勵人心）                      │
│  精選故事 3：新手的故事（降低門檻）                          │
│                                                             │
│  「XX 人已經分享了他們的故事」                               │
└─────────────────────────────────────────────────────────────┘
```

## 四、社群推廣策略（Phase 1）

### 4.1 FB 攀岩社團策略

```
目標社團：
├─ 台灣攀岩資訊站
├─ 攀岩交流區
├─ 龍洞攀岩
├─ 各岩館社團
└─ 戶外攀岩相關社團

發文策略（不是打廣告，是分享內容）：

❌ 錯誤：「我做了一個網站，歡迎來註冊」
✓ 正確：「整理了一些攀岩者的故事，來看看大家怎麼入坑的」

發文範例：
┌─────────────────────────────────────────────────────────────┐
│  【攀岩者入坑故事募集】                                      │
│                                                             │
│  最近在整理攀岩者的故事，發現大家入坑的方式超多元：          │
│                                                             │
│  「被室友拉去，本來只是去看看...」                          │
│  「在 Netflix 看了攀岩紀錄片...」                           │
│  「路過岩館覺得很酷就進去了...」                            │
│                                                             │
│  你是怎麼開始攀岩的？                                       │
│  歡迎來這裡分享你的故事 👉 [連結]                           │
│                                                             │
│  已經有 XX 人分享了他們的故事                               │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 LINE 群組策略

```
比 FB 更私密，需要更謹慎

策略：
1. 先在群組裡建立信任（不是一進去就發連結）
2. 透過群組認識的人，私訊邀請
3. 等有一定用戶後，請用戶自己分享

話術：
「欸我最近在整理攀岩故事，你的 [某個經歷] 很有趣，
 要不要來這裡寫一下？」
```

### 4.3 內容分享策略

**讓用戶幫你分享，比你自己分享有效**

```
分享觸發點設計：

1. 完成人物誌後
   → 「你的攀岩故事已建立！分享給朋友看看？」
   → 提供一鍵分享按鈕

2. 收到回應時
   → 「阿華回應了你的故事，來看看他說什麼」
   → 用戶可能會分享這個互動

3. 達成成就時
   → 「恭喜完成 100 條路線！」
   → 提供分享圖片

分享預覽優化：
┌─────────────────────────────────────────────────────────────┐
│  [用戶的大頭照或攀岩照]                                      │
│                                                             │
│  NobodyClimb                                                │
│  「攀岩教會我不用跟別人比較」                               │
│  — 小美的攀岩故事                                           │
└─────────────────────────────────────────────────────────────┘
```

## 五、岩館合作策略（Phase 2）

### 5.1 合作價值主張

**對岩館的好處**：
```
1. 會員經營工具
   → 「建立你們岩館的攀岩者社群」
   → 會員可以在平台上互動、認識彼此

2. 內容行銷
   → 「你們的會員故事可以展示在岩館頁面」
   → 增加岩館的人情味和故事性

3. 活動宣傳
   → 「可以在平台上發布岩館活動」
   → 觸及攀岩者社群
```

### 5.2 合作方式

```
輕量合作：
├─ 岩館提供 QR Code 展示位置
├─ 我們提供「掃描寫故事」的活動設計
└─ 雙方在社群互相宣傳

中度合作：
├─ 岩館協助邀請會員加入
├─ 在岩館辦「故事分享活動」
└─ 岩館頁面在平台上建立

深度合作：
├─ 岩館的會員系統串接
├─ 岩館專屬社群功能
└─ 數據分析合作
```

### 5.3 岩館活動設計

```
【故事牆活動】

地點：合作岩館
時間：週末下午

流程：
1. 現場設置「故事牆」海報
2. 岩友用手機掃 QR Code
3. 回答一個問題：「你是怎麼開始攀岩的？」
4. 回答會顯示在現場螢幕上
5. 當天最多人回應的故事可獲得小禮物

目標：
├─ 當天新增 30+ 用戶
├─ 產生 30+ 則故事內容
└─ 讓岩館會員互相認識
```

## 八、常見問題與對策

### Q1：種子用戶不寫怎麼辦？

```
原因診斷：
├─ 不知道寫什麼 → 提供具體範例
├─ 沒時間 → 降低到「一句話就好」
├─ 覺得沒必要 → 強調「你的故事可以幫助別人」
└─ 忘記了 → 定期提醒

對策：
1. 主動幫他開頭：「我幫你寫第一句：那天是...」
2. 提供填空式模板
3. 約時間一起寫（視訊陪伴）
```

### Q2：用戶來了但不互動怎麼辦？

```
原因診斷：
├─ 不知道要互動 → 加強引導
├─ 沒有想回應的內容 → 改善內容品質
├─ 互動成本太高 → 降低互動門檻
└─ 沒有看到別人互動 → 展示互動氛圍

對策：
1. 在故事下方加「快速回應」按鈕
2. 推播「這個故事很多人有共鳴，你也是嗎？」
3. 展示「XX 人回應了這個故事」
```

### Q3：內容品質參差不齊怎麼辦？

```
對策：
1. 精選機制：首頁只展示優質內容
2. 引導改善：提供「讓故事更好」的提示
3. 範例展示：讓用戶知道「好故事長什麼樣」
4. 不公開懲罰：低品質內容降低曝光，但不刪除
```

### Q4：成長停滯怎麼辦？

```
診斷方向：
├─ 獲客問題：沒有新用戶進來
├─ 留存問題：用戶來了就走
├─ 活躍問題：用戶不回來
└─ 貢獻問題：沒人產出內容

對策：
1. 回到用戶訪談，了解真正原因
2. 檢視每個環節的轉換率
3. 小規模測試新策略
4. 不要一次改太多東西
```

## 十、總結

### 冷啟動成功的關鍵

```
1. 先有內容，再有用戶
   → 種子用戶不是「用戶」，是「內容生產者」

2. 每個故事都要有回應
   → 初期你要親自確保這件事

3. 降低門檻比增加功能重要
   → 「寫一句話」比「完整人物誌」優先

4. 社會證明很重要
   → 「XX 人已經分享」比「歡迎註冊」有效

5. 口碑 > 廣告
   → 一個人真心分享 > 十次官方發文
```

### 執行心態

```
冷啟動不是一次性的事，是持續的努力。

第一個月：手工做事（一個一個邀請、一個一個回應）
第二個月：找到模式（什麼有效、什麼沒用）
第三個月：開始規模化（用系統取代手工）

記住：每一個真正投入的用戶，都是靠你親自邀請、親自互動換來的。
```
