# 觸發點設計（Hook Design）

> **目標**：設計讓用戶養成使用習慣的觸發機制
> **框架**：基於 Nir Eyal 的 Hook Model（觸發 → 行動 → 變動獎勵 → 投入）

---

## 一、Hook Model 概述

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│     ┌──────────┐                      ┌──────────┐         │
│     │  觸發    │ ──────────────────▶ │  行動    │         │
│     │ Trigger  │                      │ Action   │         │
│     └──────────┘                      └────┬─────┘         │
│          ▲                                 │                │
│          │                                 ▼                │
│     ┌──────────┐                      ┌──────────┐         │
│     │  投入    │ ◀──────────────────  │  獎勵    │         │
│     │Investment│                      │ Reward   │         │
│     └──────────┘                      └──────────┘         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

| 階段 | 說明 | NobodyClimb 應用 |
|------|------|------------------|
| **觸發** | 驅動用戶採取行動的提示 | 通知、Email、攀岩後想記錄 |
| **行動** | 用戶為了獲得獎勵而做的事 | 回應故事、上傳紀錄、按讚 |
| **獎勵** | 讓用戶滿足的回報（要有變動性） | 被回應、獲得讚、發現有趣內容 |
| **投入** | 用戶投入的時間/內容/社交 | 寫的故事、建立的連結、累積的紀錄 |

由平台主動發送給用戶的提示

```
│  ├─ 本週熱門故事：「攀岩教會我的事」有 23 則新回應          │
```

**通知文案設計原則**：

| 原則 | 範例 |
|------|------|

| 製造好奇 | ❌「有新內容」→ ✅「12 人回應了『攀岩對你的意義』，最熱門的是...」 |
| 行動導向 | ❌「歡迎回來」→ ✅「回應阿華的留言？」 |
| 社交證明 | ❌「來寫故事」→ ✅「本週有 89 人分享了攀岩故事」 |

#### 類型 C：情境觸發

基於用戶所在情境的觸發

```
情境 1：看到朋友分享
──────────────────
觸發時機：用戶從社群連結進來
觸發內容：「小美分享了這個故事，你也有類似經歷嗎？」

──────────────────
觸發時機：用戶讀完一則人物誌

目標行動：回應故事

情境 3：新用戶註冊後
──────────────────
觸發時機：完成註冊但尚未填寫故事
觸發內容：「30 秒完成第一個故事，分享你的攀岩起點」
目標行動：填寫核心故事第一題
```

### 2.2 內部觸發（Internal Triggers）

用戶自己產生的動機，不需要外部提示

```
┌─────────────────────────────────────────────────────────────┐
│  🧠 內部觸發培養                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  目標：讓用戶在特定情境自動想到 NobodyClimb                  │
│                                                             │
│  情境 → 內部觸發 → 行動                                      │
│                                                             │
│  「看到別人的故事有共鳴」                                    │
│   → 想表達認同、想分享經驗                                   │
│   → 打開 NobodyClimb 回應故事                                │
│                                                             │
│  「在岩場遇到不認識但很厲害的人」                            │
│   → 好奇這個人是誰                                          │
│   → 搜尋 NobodyClimb 看他的人物誌                            │
│                                                             │
│  「想起自己的攀岩故事」                                      │
│   → 想記錄、想分享                                           │
│   → 更新 NobodyClimb 人物誌                                  │
│                                                             │
│  「無聊滑手機」                                              │
│   → 想看攀岩相關內容                                         │
│   → 打開 NobodyClimb 看故事                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**培養內部觸發的方法**：
1. 重複的外部觸發 → 形成習慣
2. 創造獨特價值（只有這裡有的內容）
3. 建立情感連結（社群歸屬感）

## 三、行動設計（降低摩擦）

### Fogg 行為模型

```
行動 = 動機 × 能力 × 觸發

當觸發出現時，用戶需要有足夠的動機和能力才會行動。
```

### 降低行動門檻

| 行動 | 現有摩擦 | 優化方案 |
|------|---------|---------|
| 回應故事 | 要想很久才知道寫什麼 | 提供快速回應按鈕：「我也是」「+1」「說得好」 |
| 寫人物誌 | 34 題太多 | 一次只問一題，漸進解鎖（MVP 聚焦 3 題核心故事） |
| 按讚 | 沒問題 | 維持簡單 |
| 追蹤他人 | 沒問題 | 維持簡單 |
| 分享故事 | 不知道怎麼分享 | 一鍵分享到 FB/LINE，自動產生分享圖 |

### 快速回應設計

```
┌─────────────────────────────────────────────────────────────┐
│  @小美：「攀岩對我來說是逃離日常的出口」                      │
│                                                             │
│  快速回應：                                                  │
│  ┌────────┬────────┬────────┬────────┐                     │
│  │ 😂     │ 🙌     │ 💪     │ 🤝     │                     │
│  │ 我也是 │ 說得好 │ +1     │ 同感   │                     │
│  └────────┴────────┴────────┴────────┘                     │
│                                                             │
│  或寫下你的想法：                                            │
│  [ __________________________________ ] [ 送出 ]            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 四、變動獎勵設計（Variable Reward）

### 為什麼需要「變動」？

固定獎勵會讓人麻木，變動獎勵讓人期待。

```
固定獎勵：每次發文都得到 10 點 → 很快無感
變動獎勵：不知道會有多少人回應 → 持續期待
```

### 三種獎勵類型

#### 類型 1：社群獎勵（Tribe）

來自他人的認可和連結

```
┌─────────────────────────────────────────────────────────────┐
│  社群獎勵設計                                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【被認可】                                                  │
│  • 故事被按讚 → 「阿華、小美等 12 人覺得有共鳴」             │
│  • 故事被回應 → 看到別人分享的經驗，產生連結                │
│  • 被追蹤 → 「又有人對你感興趣了」                          │
│  • 回應被按讚 → 「你的回應引起了 5 人共鳴」                 │
│                                                             │
│  【歸屬感】                                                  │
│  • 發現同類 → 「有 89 人也是被朋友拉去攀岩的」              │
│  • 社群討論 → 在故事回應區看到熱烈討論                      │
│  • 被記住 → 週年提醒、成就回顧                              │

│  【變動性】                                                  │
│  • 不知道誰會回應                                           │
```

## 五、投入設計（Investment）

### 投入創造價值

用戶投入越多，離開成本越高。

```
投入類型：

1. 內容投入
   └─ 寫的故事、上傳的照片、累積的紀錄
   └─ 這些內容「只存在於這裡」

2. 社交投入
   └─ 建立的追蹤關係、互動的朋友
   └─ 這些連結「無法帶走」

3. 聲望投入
   └─ 累積的讚數、獲得的徽章、建立的信譽
   └─ 這些地位「重新開始很難」

4. 學習投入
   └─ 熟悉的介面、掌握的技巧
   └─ 這些經驗「需要時間」
```

### 投入驅動下一次觸發

```
投入 → 儲存價值 → 創造觸發

例子：
用戶寫了一個故事（投入）
→ 故事被儲存、可被回應（儲存價值）
→ 有人回應時發通知（創造觸發）
→ 用戶回來看回應（新的行動）
→ 循環繼續
```

## 六、習慣養成時程

### 21 天習慣養成計劃

```
第 1-3 天：密集觸發
──────────────────────
目標：讓用戶體驗核心價值
觸發：
• Day 1：歡迎 + 引導完成第一個故事
• Day 2：「你的故事收到了第一個回應」（確保有回應）
• Day 3：「發現跟你類似的攀岩者」

第 4-7 天：建立連結
──────────────────────
目標：創造社交連結
觸發：
• Day 4：推薦追蹤對象
• Day 5：「你追蹤的人更新了」
• Day 6-7：週末攀岩提醒

第 8-14 天：培養習慣
──────────────────────
目標：形成使用模式
觸發：
• 週日晚上：週報 + 記錄提醒
• 有互動時：即時通知
• 無互動時：內容推薦

第 15-21 天：強化行為
──────────────────────
目標：讓習慣根深蒂固
觸發：
• 減少頻率，觀察主動回訪
• 保持核心通知（社交互動）
• 加入驚喜元素（週年、成就）
```

## 七、通知系統規格

### 通知類型與頻率

| 類型 | 優先級 | 頻率限制 | 可關閉 |
|------|--------|---------|--------|
| 有人回應你 | 高 | 即時 | 否 |
| 有人按讚 | 中 | 彙整（每 3 小時） | 是 |
| 有人追蹤 | 中 | 即時 | 是 |
| 回應被按讚 | 中 | 彙整（每 3 小時） | 是 |
| 追蹤者更新 | 低 | 每日彙整 | 是 |
| 內容推薦 | 低 | 每週 1 次 | 是 |
| 故事完成提醒 | 低 | 每週 1 次 | 是 |

### 通知渠道

| 渠道 | 使用時機 | 注意事項 |
|------|---------|---------|
| App Push | 高優先級、即時性 | 不可過於頻繁 |
| Email | 低優先級、彙整式 | 週報形式 |
| Web Push | 桌面用戶 | 取得許可 |
| In-App | 進入 App 時 | 通知中心 |

### 通知文案模板

```
【有人回應】
標題：阿華回應了你的故事
內容：「我也是被朋友拉去的！」— 點擊查看完整回應

【有人追蹤】
標題：小美開始追蹤你了
內容：看看她的攀岩故事 →

【週報】
標題：你的 NobodyClimb 週報
內容：本週 12 人對你的故事有共鳴 ✨

【記錄提醒】
標題：這週末去攀岩了嗎？
內容：記錄一下，讓未來的你回味這一刻 📸
```

## 八、反模式警示

### 避免這些做法

| 反模式 | 問題 | 後果 |
|--------|------|------|
| 過度通知 | 每個小動作都發通知 | 用戶關閉通知 |
| 假緊急 | 「你的故事快要過期了」 | 失去信任 |
| 操控性文案 | 「你確定要離開嗎？」 | 反感 |
| 無用通知 | 「歡迎回來」 | 被忽略 |
| 固定獎勵 | 每次都一樣的回饋 | 無感 |

### 健康的觸發原則

```
✓ 觸發要有價值：點進去確實有用戶想看的
✓ 觸發要尊重用戶：提供關閉選項
✓ 觸發要誠實：不誇大、不製造假緊急
✓ 觸發要適量：寧缺勿濫
✓ 觸發要個人化：根據用戶行為調整
```
