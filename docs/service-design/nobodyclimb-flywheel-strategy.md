# NobodyClimb 雙核心飛輪效應規劃

## 設計理念

NobodyClimb 的飛輪設計採用「行動派」與「故事派」雙核心架構。此設計的核心理念是：**讓不同類型的攀岩者都能找到參與入口，並透過互相強化形成正向循環。**

飛輪效應的關鍵：**每個用戶行為都應該強化其他行為，形成自我增強的循環。**

---

## 兩種用戶類型的入口

| 用戶類型 | 入口 | 飛輪路徑 |
|---------|------|---------|
| **行動派** | 完攀路線 → 上傳紀錄 | 完攀紀錄 → 路線豐富 → 被發現 → 社群連結 |
| **故事派** | 閱讀故事 → 產生共鳴 | 回應故事 → 互動討論 → 想寫自己的 → 豐富人物誌 |

---

## 飛輪交會點：人物誌（個人中心）

兩個飛輪最終都匯聚到「人物誌」，這是用戶的攀岩身份與成長紀錄：

```
完攀紀錄 ──→ 攀登履歷 ──┬──→ 完整的人物誌 ──→ 被更多人發現/認可
                        │
故事回應 ──→ 故事內容 ──┘
```

人物誌作為交會點的設計意義在於：無論用戶從哪個入口進入平台，最終都會在這裡留下可視化的攀岩痕跡，而這些痕跡又會吸引更多人參與，形成網絡效應。

## 核心價值主張對照

| 對象 | 行動型飛輪價值 | 故事型飛輪價值 |
|------|---------------|---------------|
| **攀岩者** | 記錄成長軌跡，建立可視化攀登履歷 | 分享故事獲得共鳴，找到志同道合的人 |
| **社群** | 互相幫忙紀錄，豐富路線 Beta | 話題討論，建立情感連結與協作網絡 |
| **內容** | 累積路線資料與攀登參考 | 豐富互動內容，啟發更多人 |

這個價值主張設計呼應了自我決定理論（Self-Determination Theory）的三大心理需求：行動型飛輪主要滿足「勝任感」（Competence），故事型飛輪主要滿足「關係感」（Relatedness），而兩者都賦予用戶「自主性」（Autonomy）。

## 為什麼 NobodyClimb 需要雙核心？

### 1. 降低參與門檻

不是每個人都經常完攀新路線，但每個人都有故事可以分享、可以回應。雙核心設計讓「還沒有精彩完攀紀錄的用戶」也能找到參與的方式，而不會因為「還不夠強」而離開平台。

### 2. 不同時機的參與

完攀紀錄需要用戶有實際攀登才能產出內容，這通常發生在特定的攀岩日；而故事回應則可以在任何零碎時間進行，例如通勤時看看別人的故事、睡前給一個共鳴的回應。這樣的設計讓用戶可以在不同的生活情境中持續與平台互動。

### 3. 互相導流

兩個飛輪之間會產生自然的用戶流動：

- **行動派 → 故事派**：看到厲害的完攀 → 好奇這個人的故事 → 看人物誌 → 想了解更多
- **故事派 → 行動派**：看到有趣的故事 → 好奇這個人爬過什麼 → 看完攀紀錄 → 想去挑戰同樣路線

這種交互導流讓平台的整體活躍度提升，也讓用戶的參與深度逐漸加深。

## 飛輪啟動的關鍵機制

這個雙核心飛輪一旦轉起來，無論用戶是「行動派」還是「故事派」，都能找到參與的入口，並且互相強化整個生態系統的價值：

| 階段 | 行動型飛輪的動能 | 故事型飛輪的動能 | 交互效應 |
|------|----------------|----------------|---------|
| **啟動期** | 用戶上傳第一筆完攀紀錄 | 用戶填寫第一個故事欄位 | 故事吸引人來看完攀紀錄 |
| **累積期** | 持續記錄累積攀登履歷 | 故事收到回應產生互動 | 完攀展示吸引更多人想認識 |
| **擴散期** | 路線頁被分享、SEO 提升 | 一句話系列形成話題討論 | 成功案例帶動新用戶兩邊參與 |
| **自轉期** | 用戶主動幫朋友紀錄 | 資深用戶主動回應新手故事 | 平台產生自主的內容與連結 |

### 啟動期的關鍵動作

在飛輪剛開始轉動時，需要人工介入來確保「第一圈」能順利轉動。具體做法包括：
- 邀請核心攀岩者上傳種子內容
- 團隊成員主動給予故事回應
- 策劃線上活動讓用戶有「第一次紀錄」的契機

### 自轉期的指標

當平台達到自轉期時，會出現以下現象：
- 用戶自發邀請朋友加入
- 資深用戶主動幫助新手、回應故事
- 人物誌被分享到平台外部（如社群媒體、求職時展示）
- 攀岩相關單位開始主動詢問合作可能

## 與商業模式的對接

這個飛輪設計也直接對接 NobodyClimb 的營收模式：

### 免費版（Freemium）入口

兩個核心功能的基礎版本都免費，讓用戶能輕鬆進入飛輪。這包括：
- 基本的完攀紀錄功能
- 人物誌故事欄位
- 基礎版攀登統計

### 付費轉換點

| 飛輪類型 | 付費價值 | 對應方案 |
|---------|---------|---------|
| 行動型飛輪 | 進階統計分析、無限照片儲存、年度回顧報告 | 個人進階版訂閱（B2C） |
| 故事型飛輪 | 岩館社群管理、會員數據追蹤、活動管理 | 岩館企業服務（B2B） |

### 飛輪與付費轉換的關係

飛輪設計的核心精神是：**用戶在免費版累積的價值越多，付費的意願就越高。** 當用戶的人物誌累積了大量完攀紀錄和故事，他們會想要更進階的分析功能；當岩館看到平台上的社群活躍度，會想要將這樣的機制引入自己的會員經營。

這樣的設計讓「用戶價值的累積」與「付費意願的提升」形成正向循環，而非單純依賴行銷推廣來驅動成長。

## 風險與對策

| 風險 | 可能影響 | 對策 |
|------|---------|------|
| 冷啟動問題 | 初期沒人上傳，路線頁空蕩蕩 | 種子內容策略：邀請核心攀岩者先上傳 |
| 隱私顧慮 | 用戶不想被標記 | 可關閉被標記功能，需確認才顯示 |
| 資料品質 | 錯誤的 Beta、假冒的完攀 | 社群檢舉機制、信譽系統 |
| 過度遊戲化 | 為了數字而失去真實性 | 強調故事價值而非數量 |
| 內容兩極化 | 只有高手分享，新手不敢參與 | 設計新手友善的參與方式（如故事回應） |

## 總結

NobodyClimb 的雙核心飛輪設計，本質上是在回答一個問題：**如何讓不同類型的攀岩者都能在平台上找到價值，並且讓他們的參與成為其他人的養分？**

透過「行動型飛輪」與「故事型飛輪」的互相補強，我們不僅能降低用戶的參與門檻、提高平台的活躍度，更能讓每一位攀岩者的旅程都成為整個生態系統的正向資產。

當這個飛輪開始自轉，NobodyClimb 就能從「需要推廣的產品」轉變為「自然成長的攀岩社群生態系」。
