## 核心問題與解法

### 問題診斷
> 「很多人看，很少人寫」的根本原因是：**用戶覺得自己沒什麼特別**

這不是功能問題，而是心理障礙。

### 解法架構

```
降低門檻              建立信心              形成習慣
   │                    │                    │
   ▼                    ▼                    ▼
引導式問答           展示素人故事          確保每則故事
（3個簡單問題）      （平凡中的不平凡）    都有人回應
   │                    │                    │
   └────────────────────┴────────────────────┘
                        │
                        ▼
              「我的故事也值得分享」
```

## MVP 策略

本次 MVP 聚焦驗證 **故事回應飛輪**：

- **IN**: 核心故事功能、故事回應、基礎通知
- **OUT**: 完攀紀錄、進階傳記、擴展飛輪
- **驗證假設**: 「收到回應會讓用戶更願意分享」

詳見 [MVP 功能範圍](./mvp-scope.md)
