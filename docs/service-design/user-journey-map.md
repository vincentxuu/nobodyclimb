# 用戶旅程地圖（User Journey Map）

> **目標**：完整描繪用戶從發現到成為活躍貢獻者的全旅程
> **背景**：主要流量來源為社群分享，核心問題是「看得人多，寫的人少」

---

## 一、用戶角色定義（Persona）

### 主要角色：週末攀岩者

```
姓名：小美（代表 70% 的目標用戶）
年齡：28 歲
攀岩年資：1-3 年
程度：5.10a - 5.11a
頻率：每週 1-2 次（週末為主）

特徵：
- 有正職工作，攀岩是休閒
- 有固定的攀岩夥伴
- 會看 YouTube 攀岩影片
- 在 FB/IG 追蹤攀岩相關帳號
- 拍了很多攀岩照片但沒整理

心理狀態：
- 享受攀岩但不追求高難度
- 覺得自己「不夠厲害」不值得記錄
- 對社群有興趣但不主動發文
- 喜歡看別人的故事
```

### 次要角色

| 角色 | 特徵 | 佔比 |
|------|------|------|
| 新手小白 | 剛開始攀岩 < 6 個月，充滿熱情但技術生疏 | 15% |
| 進階攀岩者 | 5.12+ 程度，有戶外經驗，可能是內容貢獻者 | 10% |
| 攀岩教練/老手 | 資深攀岩者，有教學經驗，社群意見領袖 | 5% |

## 三、各階段詳細分析

### 階段 1：發現（Awareness）

**場景**：朋友在 FB/IG/LINE 分享了一個連結

```
情境舉例：
「欸你看這個網站有龍洞的路線資料」
「這個人的攀岩故事寫得很好笑」
「我在這個網站建了人物誌，你也來」
```

| 維度 | 內容 |
|------|------|
| **接觸點** | FB 貼文、IG 限動、LINE 群組、口耳相傳 |
| **用戶行為** | 點擊連結、進入網站 |
| **用戶想法** | 「這是什麼？」「看起來跟攀岩有關」 |
| **情緒** | 😐 好奇但警戒 |
| **成功指標** | 連結點擊率（CTR） |

**設計機會**：
- 分享連結的預覽圖（OG Image）要吸引人
- 標題要清楚傳達價值
- 社群分享時自動帶入吸引人的文案

**分享預覽優化**：
```
┌─────────────────────────────────────────┐
│  [精選人物誌照片]                        │
│                                         │
│  NobodyClimb                            │
│  「攀岩教會我的事：不用跟別人比較」       │
│  小美的攀岩故事 — 156 人說「我也是」      │
└─────────────────────────────────────────┘
```

---

### 階段 2：探索（Exploration）

**場景**：用戶已經進入網站，開始瀏覽

| 維度 | 內容 |
|------|------|
| **接觸點** | 首頁、人物誌頁面、路線頁面、文章 |
| **用戶行為** | 滑動瀏覽、點擊內容、閱讀故事 |
| **用戶想法** | 「這些人的故事好有趣」「原來還有這種網站」 |
| **情緒** | 🙂 有趣、想多看 |
| **成功指標** | 頁面停留時間、瀏覽頁數、跳出率 |

**用戶內心 OS**：
```
「這個人的故事好好笑」 → 繼續看
「原來很多人也是被朋友拉去的」 → 產生共鳴
「他們好像也沒有很厲害」 → 降低門檻感
「我好像也可以寫...」 → 種下種子
```

**關鍵設計**：
- 展示「普通人」的故事（參考 ux-first-mile.md）
- 適時出現「你也可以」的 CTA
- 不要太早強迫註冊

**探索階段的引導**：
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  [用戶正在看某個人物誌]                                      │
│                                                             │
│  ...小美的故事...                                           │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  💬 23 人回應了這個故事                                     │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 想回應嗎？                                          │   │
│  │                                                      │   │
│  │ [ 登入回應 ]  或  [ 繼續瀏覽 ]                       │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 階段 3：註冊（Registration）

**場景**：用戶決定建立帳號

| 維度 | 內容 |
|------|------|
| **接觸點** | 註冊頁面、社群登入 |
| **用戶行為** | 填寫資料、驗證信箱、設定密碼 |
| **用戶想法** | 「希望不要太麻煩」「資料會被怎麼用？」 |
| **情緒** | 😊 期待但有點煩 |
| **成功指標** | 註冊完成率、社群登入比例 |

**流失原因**：
- 步驟太多
- 要求太多資料
- 沒有社群登入選項
- 驗證信件跑到垃圾郵件

**優化建議**：
```
極簡註冊流程：

1. 選擇登入方式
   [ Google 登入 ] [ Facebook 登入 ] [ Email 註冊 ]

2. 取一個暱稱
   「在攀岩圈，大家怎麼叫你？」
   [ 輸入暱稱 ]

3. Done! 開始回答第一個問題（見 ux-first-mile.md）
```

---

### 階段 4：首次體驗（First Experience）⚠️ 關鍵階段

**場景**：剛註冊完，進入個人頁面

| 維度 | 內容 |
|------|------|
| **接觸點** | 個人頁面、人物誌編輯、引導流程 |
| **用戶行為** | 看到空白頁面、考慮是否填寫 |
| **用戶想法** | 「要填這麼多？」「我的故事有什麼好寫的」 |
| **情緒** | 🤔 猶豫、退縮 |
| **成功指標** | 核心故事完成率、首日填寫率 |

**這是最大流失點！**

```
流失心理路徑：
看到空白頁面 → 不知道要寫什麼 → 覺得自己沒什麼好寫 → 離開

期望心理路徑：
看到一個問題 → 這個我可以回答 → 寫完有成就感 → 想繼續
```

**關鍵設計**（詳見 ux-first-mile.md）：
- 不要給空白表單，給引導式問題
- 一次只問一題
- 提供範例和提示
- 即時展示「寫完的樣子」

---

### 階段 5：回訪（Return）

**場景**：用戶離開後，什麼時候會再回來？

| 維度 | 內容 |
|------|------|
| **接觸點** | 通知、Email、社群分享、搜尋 |
| **用戶行為** | 收到通知點進來、看到朋友分享、主動想到 |
| **用戶想法** | 「有人回應我了」「這個人更新了」 |
| **情緒** | 😊 被記得、有連結感 |
| **成功指標** | 7 日回訪率、30 日回訪率、DAU/MAU |

**回訪觸發點**：
```
被動回訪：
- 有人回應了你的故事 → 通知
- 有人追蹤了你 → 通知
- 你追蹤的人更新了 → 通知
- 週報 Email → 「本週社群精選」

主動回訪：
- 去岩場攀岩後想記錄
- 看到朋友分享了連結
- 想找某條路線資料
```

**設計機會**：
- 通知系統（詳見 hook-design.md）
- Email 週報
- 「記錄提醒」（週末晚上）

---

### 階段 6：貢獻（Contribution）

**場景**：用戶從「看的人」變成「寫的人」

| 維度 | 內容 |
|------|------|
| **接觸點** | 個人頁面、路線頁面、照片上傳 |
| **用戶行為** | 寫故事、上傳照片、回應他人、分享 Beta |
| **用戶想法** | 「我想記錄這次」「這個我可以回答」 |
| **情緒** | 🥳 成就感、歸屬感 |
| **成功指標** | 內容產出率、貢獻頻率、內容品質 |

**轉化關鍵**：
```
觸發點：
- 完成了一條很難的路線（想炫耀）
- 有人問了一個我知道答案的問題（想幫忙）
- 看到別人的故事很有趣（想回應）
- 拍了一張很滿意的照片（想分享）

阻力：
- 不知道怎麼開始
- 覺得自己寫得不好
- 上傳流程太麻煩
- 沒有時間
```

**設計機會**：
- 降低首次貢獻門檻（一句話也算）
- 快速上傳流程
- 提供範本和提示
- 即時正向回饋

```

   │                          ╭───╯

   │      ╭─╯ ╲           ╭─╯
```

**情緒低點分析**：

| 低點 | 原因 | 對策 |
|------|------|------|
| 首次體驗 | 看到空白頁面不知所措 | 引導式問答、降低門檻 |
| 等待回應 | 寫了但沒人回應 | 確保種子內容有回應、AI 輔助互動 |
| 內容被忽視 | 貢獻了但沒有正向回饋 | 通知系統、即時互動 |

## 五、關鍵時刻（Moments of Truth）

### MOT 1：第一次看到別人的故事

```
成功：「原來很多人跟我一樣」→ 產生共鳴
失敗：「這些人好厲害，我不配」→ 離開
```

**設計要點**：首頁展示「普通人」的故事

### MOT 2：註冊後看到的第一個畫面

```
成功：「這個問題我可以回答」→ 開始填寫
失敗：「要填這麼多東西」→ 離開
```

**設計要點**：一次只問一個問題

### MOT 3：第一次收到回應

```
成功：「有人看到我了！」→ 想寫更多
失敗：「寫了都沒人理」→ 放棄
```

**設計要點**：確保早期用戶都能收到回應

### MOT 4：完攀路線後想記錄

```
成功：「記錄流程好簡單」→ 養成習慣
失敗：「太麻煩了算了」→ 不記錄
```

**設計要點**：極簡上傳流程、事後提醒

## 八、衡量指標

### 漏斗指標

```
發現 → 探索：連結點擊率（目標 > 5%）
     │
     ▼
探索 → 註冊：訪客註冊轉換率（目標 > 10%）
     │
     ▼
註冊 → 首次體驗：核心故事完成率（目標 > 40%）
     │
     ▼
首次體驗 → 回訪：7 日回訪率（目標 > 30%）
     │
     ▼
回訪 → 貢獻：30 日內容產出率（目標 > 20%）
```

### 體驗指標

| 指標 | 定義 | 目標 |
|------|------|------|
| NPS | 淨推薦分數 | > 50 |
| 任務完成率 | 核心任務成功完成比例 | > 80% |
| 首次價值時間 | 註冊到看到價值的時間 | < 3 分鐘 |
| 首次貢獻時間 | 註冊到首次貢獻的時間 | < 7 天 |
