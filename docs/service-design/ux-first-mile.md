# 用戶首次體驗設計（First Mile UX）

> **核心問題**：用戶覺得「自己沒什麼特別」，所以不寫內容
> **設計目標**：降低心理門檻，讓每個人都覺得自己的故事值得被記錄

---

## 一、問題診斷

### 現況分析

```
用戶行為矛盾：
┌─────────────────────────────────────────┐
│  愛看別人的人物誌  ≠  自己卻不寫       │
└─────────────────────────────────────────┘

心理機制：
「我又不是什麼厲害的攀岩者」
「我的故事很普通，誰要看」
「等我爬到 5.12 再來寫」
「我沒有什麼值得分享的」
```

### 這不是功能問題，是心理門檻

| 錯誤假設 | 實際問題 |
|---------|---------|
| 用戶不知道有這功能 | 用戶知道，但覺得不適合自己 |
| 功能太複雜 | 功能不複雜，但「開始」很難 |
| 缺乏誘因 | 誘因不是問題，自我價值認同才是 |

---

## 二、設計原則

### 原則 1：重新定義「值得分享」

**From**：只有厲害的人才值得被記錄
**To**：每個人與攀岩的故事都獨一無二

**設計手法**：
- 展示「普通人」的故事，而非只展示強者
- 強調「過程」而非「成就」
- 使用包容性語言

### 原則 2：從最小承諾開始

**From**：請填寫你的人物誌（34 個欄位）
**To**：用一句話回答這個問題

**設計手法**：
- 漸進式揭露（Progressive Disclosure）
- 降低首次行動的成本
- 讓用戶先「嚐到甜頭」

### 原則 3：即時正向回饋

**From**：寫完等人來看（被動）
**To**：寫完立刻看到價值（主動）

**設計手法**：
- 即時視覺化回饋
- 明確的進度感
- 社群驗證（Social Proof）

## 四、人物誌填寫體驗優化

### 4.1 問題：34 題太多

現有 34 個故事欄位（3 核心 + 31 進階）對新用戶是認知負擔。

### 4.2 解法：三層漸進式結構

```
第一層：1 題（註冊時已完成）
  └─ 你是怎麼開始攀岩的？

第二層：3 題（核心故事）
  ├─ 你與攀岩的相遇（詳細版）
  ├─ 攀岩對你的意義
  └─ 給剛開始的自己一句話

第三層：31 題（進階故事）
  └─ 在個人頁面以「解鎖更多」方式呈現
```

### 4.3 UI 設計：引導式填寫

## 五、降低「特別感」門檻的具體策略

### 策略 1：展示統計數據，讓用戶知道自己不孤單

```
「有 67% 的人跟你一樣，爬了超過一年還在 5.10 以下」
「89 人也是被朋友拉進來的」
「你不是唯一覺得自己『沒什麼特別』的人」
```

### 策略 2：使用「我也是」的社群驗證

在故事回應區，刻意設計「+1」或「我也是」的快速回應：

```
┌─────────────────────────────────────────────────────────────┐
│  @小明：「第一次戶外被雨淋，什麼都沒爬成」                    │
│                                                             │
│  💬 回應：                                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ [ 😂 我也是 ]  [ 🙋 +1 ]  [ 💪 後來呢？ ]            │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  已有 12 人說「我也是」                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 策略 3：弱化成就，強化故事

**避免**：
```
「最高難度：5.12a」
「總完攀數：156 條」
```

**強調**：
```
「攀岩 3 年」
「最常去的岩場：龍洞」
「攀岩風格：享受過程派」
```

### 策略 4：提供「故事範本」降低書寫門檻

```
┌─────────────────────────────────────────────────────────────┐
│  📝 你與攀岩的相遇                                          │
│                                                             │
│  不知道怎麼開始？試試這個：                                   │
│                                                             │
│  「___年___月，我第一次接觸攀岩。                            │
│   那天是___帶我去的。                                        │
│   印象最深的是___。                                          │
│   從此之後，___。」                                          │
│                                                             │
│  或者，直接用一句話：                                        │
│  ___________________________________________________        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 六、空狀態設計（Empty State）

當頁面沒有內容時，不要讓用戶看到「空蕩蕩」，而是看到「機會」。

### 6.1 個人頁面空狀態

**不要**：
```
「你還沒有任何完攀紀錄」
「開始記錄你的攀岩履歷」
```

**要**：
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  你的攀岩故事即將開始 ✨                                    │
│                                                             │
│  不需要等到爬得很厲害才開始記錄，                            │
│  每一次嘗試都是故事的一部分。                                │
│                                                             │
│  [ 寫下第一個故事 ]                                         │
│                                                             │
│  💡 89% 的人在攀岩第一年就開始記錄了                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 路線頁面空狀態

**不要**：
```
「還沒有人完攀這條路線」
```

**要**：
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  成為第一個記錄這條路線的人 🏆                              │
│                                                             │
│  你的 Beta 可以幫助後來的攀岩者                              │
│                                                             │
│  [ 我爬過這條 ]  [ 加入待爬清單 ]                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 七、文案語氣指南

### 避免使用

| 避免 | 原因 |
|------|------|
| 「建立你的攀岩履歷」 | 太正式，像在求職 |
| 「展示你的成就」 | 暗示需要有成就才能展示 |
| 「分享你的實力」 | 排斥實力不強的人 |
| 「記錄你的戰績」 | 過度競技化 |

### 建議使用

| 建議 | 原因 |
|------|------|
| 「寫下你的故事」 | 每個人都有故事 |
| 「記錄這一刻」 | 強調當下，不強調成就 |
| 「和大家聊聊」 | 平等、輕鬆 |
| 「你的攀岩日記」 | 私密、個人化 |

## 八、成功指標

| 指標 | 定義 | 目標 |
|------|------|------|
| 註冊完成率 | 開始註冊 → 完成註冊 | > 70% |
| 首題完成率 | 註冊後回答第一個問題 | > 60% |
| 核心故事完成率 | 完成 3 個核心故事 | > 40% |
| 7 日回訪率 | 註冊後 7 天內再次訪問 | > 30% |
| 首次內容發布率 | 30 天內發布任何內容 | > 25% |

## 九、總結

### 核心轉變

```
From: 「請填寫你的資料」（任務導向）
To:   「來聊聊你的故事」（對話導向）

From: 「展示你的成就」（成就門檻）
To:   「記錄你的旅程」（過程導向）

From: 「空白表單等你填」（壓力）
To:   「一個問題開始」（輕鬆）
```

### 設計核心

**讓每個用戶都覺得：「原來我也可以寫」**
